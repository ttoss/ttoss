```shell

```

## Context-Based Architecture

All components leverage the [ttoss foundation pattern](https://github.com/ttoss/ttoss/tree/main/.github/instructions/react-packages-architecture.instructions.md): configure your theme, i18n, and notifications once at the application root, and all components automatically inherit these settings without requiring component-level props.

```tsx
import { ThemeProvider } from '@ttoss/ui';
import { I18nProvider } from '@ttoss/react-i18n';
import { NotificationsProvider } from '@ttoss/react-notifications';

function App() {
  return (
    <ThemeProvider theme={myTheme}>
      <I18nProvider locale="en-US" loadLocaleData={loadLocaleData}>
        <NotificationsProvider>
          {/* All ttoss components automatically inherit configuration */}
        </NotificationsProvider>
      </I18nProvider>
    </ThemeProvider>
  );
}
```

## Key Features

- **Theme-aware**: All components use design tokens from `@ttoss/ui` and support custom themes
- **Internationalized**: Components display messages in the locale configured via `@ttoss/react-i18n`
- **Accessible**: Built following WAI-ARIA patterns with semantic HTML and keyboard navigation
- **TypeScript**: Full type safety with comprehensive type definitions
